## Task Template

When adding new tasks, use this format:

```markdown
- [ ] **TASK-XXX**: [Brief title]
  - **Status**: [Not Started | In Progress | Blocked | Done]
  - **Priority**: [High | Medium | Low]
  - **Assignee**: [Name or Unassigned]
  - **Due Date**: YYYY-MM-DD
  - **Estimated Effort**: [Hours/days]
  - **Dependencies**: 
    - TASK-YYY (must complete first)
    - TASK-ZZZ (can be parallel)
  - **Description**: 
    [Detailed description of what needs to be done]
  - **Acceptance Criteria**:
    - [ ] Criterion 1
    - [ ] Criterion 2
    - [ ] Criterion 3
  - **Related**:
    - [SPEC Section](./SPEC.md#section)
    - [Design Decision](./DESIGN.md#decision)
    - [Issue #123](https://github.com/...)
  - **Notes**:
    [Any additional context or notes]
```

## Task Status Definitions

- **Not Started**: Task identified but not yet begun
- **In Progress**: Actively working on this task
- **Blocked**: Cannot proceed due to dependency or issue
- **Review**: Completed, awaiting review
- **Done**: Completed and verified

---

## Priority Definitions

- **High**: Critical for project success, blocking other work
- **Medium**: Important but not blocking
- **Low**: Nice to have, can be deferred

---

## Estimation Guide

- **Small**: < 2 hours
- **Medium**: 2-8 hours (1 day)
- **Large**: 1-3 days
- **Epic**: > 3 days (should be broken down)

---

## Notes

- Update task status regularly
- Move completed tasks to "Completed" section
- Break large tasks into smaller ones
- Link tasks to SPEC and DESIGN docs
- Document blockers immediately
